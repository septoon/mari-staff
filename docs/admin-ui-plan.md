# План и конспект UI админки Mari

## 1. Цель
Собрать mobile-first админку салона красоты `Mari` на `React + TypeScript + Tailwind` по предоставленным скриншотам и API-спецификации (`frontend-staff.md`).

## 2. Экраны (нижний таббар)
1. `Журнал`
2. `График`
3. `Клиенты`
4. `Уведомления`
5. `Еще`

## 3. Конспект визуального стиля
- Формат: мобильный viewport, светлый фон, мягкие серые тона.
- Главный фон: светло-серый `#f3f4f6` / `#f5f6f8`.
- Основной текст: темно-графитовый `#2a3138`.
- Вторичный текст: серый `#8a919d`.
- Разделители: `#e3e6eb`.
- Акценты:
  - желтый (`#f4c900`) для активных элементов графика/кнопок;
  - голубой (`#dfe6f2`) для карточек уведомлений;
  - темный графит (`#222b33`) для переключателей дней/месяцев;
  - зеленый (`#a2dbb7`) для карточек записей в журнале.
- Радиусы: крупные (16-24px) для карточек/пилюль, средние (10-14px) для инпутов.
- Иконки: монохромные line-icons, активный таб темнее неактивных.
- Типографика:
  - жирные заголовки экранов;
  - компактные подписи в таббаре;
  - данные в списках с чёткой иерархией (имя > телефон).

## 4. Конспект по экранам

### 4.1 Экран `Еще`
- Заголовок `Еще`.
- Блок профиля пользователя.
- Пункт меню `Политика конфиденциальности`, который открывает отдельный экран.
- Блок филиала `MARi beauty`.
- Список пунктов меню с иконкой слева и шевроном справа.
- Блок: `Другие возможности MARI`.

### 4.2 Экран `Уведомления`
- Заголовок `Уведомления` и иконка действий справа.
- Секция `Записи` + ссылка `Посмотреть все`.
- Карточка уведомлений с тремя строками.
- Секция `Остальные` и состояние `Нет уведомлений`.

### 4.3 Экран `Клиенты`
- Заголовок + кнопки `меню/добавить`.
- Поисковая строка.
- Вертикальный список клиентов.
- В строке: имя, телефон, кнопка контекст-меню.

### 4.4 Экран `График`
- Заголовок `График работы`.
- Фильтр команды `Все`.
- Таблица: сотрудники по строкам, дни по столбцам, ячейки со временем.
- Кнопка `Редактировать график` (disabled-вид).
- Переключатель месяцев на темной плашке с активным жёлтым табом.

### 4.5 Экран `Журнал`
- Дата + фильтр.
- Горизонтальный список сотрудников.
- Сетка времени (часы по вертикали).
- Штрихованные слоты (недоступно).
- Карточки записей (зелёные) с временным диапазоном и данными клиента.
- Плашка дней недели над нижним таббаром.

## 5. Технический план
1. Инициализировать `CRA + TS`.
2. Подключить `Tailwind` и дизайн-токены через CSS variables.
3. Сделать общий `MobileShell`:
   - status bar;
   - контент;
   - фиксированный нижний таббар.
4. Реализовать 5 экранов как React-компоненты.
5. Добавить мок-данные, согласованные с контрактами (`clients`, `appointments`, `staff`).
6. Проверить сборку (`npm run build`).

## 6. Scope v1
- UI-реплика экранов без реальных API-запросов.
- Основа готова для интеграции endpoint'ов из `frontend-staff.md`.
